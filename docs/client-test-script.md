## Artwork to Have Ready

Please use 2 or 3 realistic customer files, not confidential production artwork:

- One transparent PNG logo or graphic.
- One JPG or PNG with small text or fine detail.
- If available, one intentionally low-resolution image so you can judge the warning.
- Recommended test size: under 20 MB per file.
- Do not use SVG, PDF, HEIC, or other formats unless the upload screen says they are supported.

## 15-Minute Test Script

### Customer flow

1. Open the customer app and upload the test artwork.
2. Confirm the artwork appears on the sheet automatically.
3. Move and resize one decal, then add or duplicate another.
4. Use **Auto-arrange** and decide whether the result is sensible.
5. Read the size, DPI, and preflight messages. Note anything unclear.
6. Check that the preview and order summary match what you expect.
7. Select **Submit for Print**.
8. On the receipt, record the project ID and confirm it lists the print PDF, proof preview, order details, and project backup.

### Admin flow

1. Open the admin review link and find the project ID from the receipt.
2. Confirm the page makes the file to print obvious.
3. Open the **Print PDF** and confirm it matches the submitted sheet.
4. Confirm the proof preview and supporting files are available.
5. Enter a reviewer name and note, then choose **Approve**, **Needs changes**, or **Reject**.
6. Confirm the review history shows the decision, reviewer, note, and time.

## What to Report

Please report anything confusing, slow, missing, or visually wrong, even if you can continue.

- **Blocker:** You cannot complete or review an order; data or files are lost; the wrong file would be printed.
- **High:** A major step fails or produces an incorrect result, but there is a workaround.
- **Medium:** The workflow works, but wording, layout, or behavior causes hesitation or repeated effort.
- **Low:** Cosmetic polish, minor wording, or a suggestion that does not impede the task.
